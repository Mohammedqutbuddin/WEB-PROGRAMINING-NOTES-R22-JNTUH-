Unit I

Client-Side Programming

HTML · CSS · JavaScript · HTML5 · Canvas · CSS3

HTML (HyperText Markup Language) is the standard language to create web pages. It uses tags — keywords inside angle brackets — to tell the browser how to display content.

<!DOCTYPE html>         <!-- Tells browser: this is HTML5 -->
<html>
  <head>                   <!-- NOT visible — metadata goes here -->
    <title>My Page</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>                   <!-- VISIBLE content goes here -->
    <h1>Hello World!</h1>   <!-- Headings: h1 (biggest) to h6 (smallest) -->
    <p>This is a paragraph.</p>
    <a href="https://google.com">Click me</a>  <!-- link -->
    <br>                      <!-- line break (self-closing) -->
    <hr>                      <!-- horizontal rule -->
    <b>Bold</b>  <i>Italic</i>  <u>Underline</u>
  </body>
</html>
🔑 Remember
  • <head> = invisible metadata | <body> = everything visible
  • Most tags need an opening and closing tag: <p>…</p>
  • Self-closing tags: <br> <hr> <img> <input>

function validate() {
  var name  = document.getElementById("name").value;
  var email = document.getElementById("email").value;

  if (name === "") {
    alert("Name is required!");
    return false;   // stops form submission
  }

    return false;
  }
  return true;      // allows submission
}

Exceptions are runtime errors. Use try-catch to handle them gracefully instead of crashing.

try {
  int result = 10 / 0;      // throws ArithmeticException!
} catch (ArithmeticException e) {
  System.out.println("Error: " + e.getMessage()); // / by zero
} catch (Exception e) {     // catches all other exceptions
  System.out.println("General error");
} finally {
  System.out.println("This ALWAYS runs"); // cleanup code here
}

JDBC, Networking & Java Beans

Database Connectivity · Sockets · TCP/UDP · RMI · Beans

JDBC lets Java programs connect and interact with relational databases like MySQL, Oracle, etc.

5 Steps to Use JDBC
  1. Load the JDBC Driver
  2. Establish a Connection
  3. Create a Statement object
  4. Execute the SQL query
  5. Process results and close connection
import java.sql.*;

public class DBConnect {
  public static void main(String[] args) throws Exception {

    // Step 1: Load Driver
    Class.forName("com.mysql.cj.jdbc.Driver");

    // Step 2: Connect to DB
    Connection con = DriverManager.getConnection(
      "jdbc:mysql://localhost:3306/mydb", "root", "password");

    // Step 3: Statement
    Statement stmt = con.createStatement();

    // Step 4: Execute query
    ResultSet rs = stmt.executeQuery("SELECT * FROM students");

    // Step 5: Process & close
    while (rs.next()) {
      System.out.println(rs.getString("name") + " | " + rs.getInt("age"));
    }
    con.close();
  }
}

JavaBean — 4 Rules (all must be followed)
  1. Must have a public no-argument constructor
  2. All properties must be private
  3. Must have public getters and setters
  4. Must implement java.io.Serializable
import java.io.Serializable;

public class StudentBean implements Serializable {
  private String name;  // private fields
  private int    age;

  public StudentBean() {}  // no-arg constructor

  public String getName()      { return name; }
  public void   setName(String n) { name = n; }
  public int    getAge()       { return age;  }
  public void   setAge(int a)   { age  = a;  }
}
RMI — Remote Method Invocation Steps
  1. Create Remote interface (extends Remote)
  2. Implement it (extends UnicastRemoteObject)
  3. Start RMI Registry (rmiregistry)
  4. Server binds object: Naming.rebind("name", obj)
  5. Client looks up: Naming.lookup("rmi://host/name")

Applets, AWT, Servlets & JSP

Java GUI · Event Handling · Web Servlets · Cookies · Sessions · JSP

Applet Lifecycle — 5 Methods (In Order!)
  1. init() — called ONCE when applet first loads (like a constructor)
  2. start() — called every time applet becomes visible
  3. paint(Graphics g) — called to draw the applet UI
  4. stop() — called when applet is hidden or minimized
  5. destroy() — called ONCE when applet is removed/closed
import java.applet.*;
import java.awt.*;
/* <applet code="MyApplet.class" width="400" height="200"></applet> */

public class MyApplet extends Applet {

  public void init()  { /* one-time setup */ }
  public void start() { /* start/resume */   }

  public void paint(Graphics g) {
    g.setColor(Color.RED);
    g.drawString("Hello Applet!", 50, 50);  // text at (50,50)
    g.fillRect(100, 100, 80, 50);           // filled rectangle
    g.drawOval(200, 100, 60, 60);           // oval/circle
  }

  public void stop()    { /* pause */   }
  public void destroy() { /* cleanup */ }
}

Servlet Lifecycle — 3 Phases
  1. init() — called ONCE when servlet is first loaded by server
  2. service() — called for EVERY request (routes to doGet or doPost)
  3. destroy() — called ONCE before servlet is unloaded
import javax.servlet.*;
import javax.servlet.http.*;
import java.io.*;

public class HelloServlet extends HttpServlet {

  public void init() {
    System.out.println("Servlet loaded!");
  }

  // Handle GET requests (e.g., clicking a link)
  protected void doGet(HttpServletRequest req, HttpServletResponse res)
      throws IOException {
    String name = req.getParameter("name");   // get form input
    res.setContentType("text/html");
    PrintWriter out = res.getWriter();
    out.println("<h1>Hello, " + name + "!</h1>");
  }

  // Handle POST requests (e.g., form submit)
  protected void doPost(HttpServletRequest req, HttpServletResponse res)
      throws IOException {
    doGet(req, res);  // delegate to doGet
  }

  public void destroy() {
    System.out.println("Servlet unloaded!");
  }
}

4 Session Tracking Techniques
  • Cookies — small data stored on client's browser
  • URL Rewriting — session ID added to every URL
  • Hidden Form Fields — session ID in hidden input
  • HttpSession — server-side session object (most common!)
// COOKIES
Cookie c = new Cookie("username", "Ravi");
c.setMaxAge(86400);      // expires in 1 day (in seconds)
response.addCookie(c);   // send to browser

// Read cookies from request
Cookie[] cookies = request.getCookies();
for (Cookie ck : cookies)
  System.out.println(ck.getName() + " = " + ck.getValue());
// HttpSession (server-side)
HttpSession session = request.getSession();  // create/get session

// Store data
session.setAttribute("user", "Ravi");
session.setAttribute("role", "admin");

// Retrieve data (on next page)
String user = (String) session.getAttribute("user");

// End session (logout)
session.invalidate();

// Session ID
String id = session.getId();  // unique session ID
